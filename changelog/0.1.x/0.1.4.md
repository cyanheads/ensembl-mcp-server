# 0.1.4 ‚Äî 2026-06-11

## Changed

- **`@cyanheads/mcp-ts-core`** `^0.9.21` ‚Üí `^0.10.6` ‚Äî framework maintenance adoption.
- **`createApp()`** ‚Äî `name` and `title` fields set explicitly to `'ensembl-mcp-server'` for identity consistency across all display surfaces.
- **`no_input` error contract** ‚Äî `ensembl_lookup_gene` and `ensembl_get_homology` now declare the missing-input case as `ValidationError` instead of `InvalidParams`, aligning with the framework's baseline validation code.
- **`Dockerfile`** ‚Äî added `APP_VERSION` build-arg and `org.opencontainers.image.version` OCI label; added `HEALTHCHECK` using a Bun-native `fetch` to `/healthz` (no curl/wget dependency).
- **`bundle` script** ‚Äî now runs `scripts/clean-mcpb.ts` after pack to prune dev deps (`mcpb clean`) and strip dependency-shipped agent docs (`node_modules/**` `skills/`, `.claude/`, `.agents/`, `SKILL.md`) that root-anchored `.mcpbignore` patterns cannot reach.
- **`lint-packaging.ts`** ‚Äî added bundle-content and identity guards: `.mcpbignore` anchoring, critical-runtime-path protection, post-bundle agent-doc content check, and `name`/`title`/`display_name` identity alignment.
- **`check-framework-antipatterns.ts`** ‚Äî added a `z.coerce.boolean()` env-flag rule (use `z.stringbool()`) and skips matches on comment lines.
- **`@types/node`** `^25.9.2` ‚Üí `^25.9.3` (dev dep).
- **Skills** ‚Äî synced to framework `^0.10.6` skill set.
