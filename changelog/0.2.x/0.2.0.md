# 0.2.0 — 2026-06-21

## Changed

- **`ensembl_get_sequence` output field `lengthInBp` → `length`** — the field carries the residue count for `type: protein`, where "bp" was wrong; renamed to the unit-neutral `length`. `format()` now labels the value by molecule type (`residues` for protein, `bp` for genomic/cdna/cds) instead of the ambiguous `bp/aa`. Nucleotide-type counts are unchanged; transcript `lengthInBp` fields (genuine base pairs) are untouched. ([#4](https://github.com/cyanheads/ensembl-mcp-server/issues/4))
- **`ensembl_get_homology` and `ensembl_lookup_gene` reject conflicting inputs** — a call providing more than one of `symbol`, `id`, `ids`, or `symbols` now fails with a `conflicting_input` `ValidationError`. Previously both tools silently resolved by an undocumented precedence (homology let `id` win; lookup ordered ids → symbols → symbol → id), contradicting the "cannot be combined" descriptions. ([#5](https://github.com/cyanheads/ensembl-mcp-server/issues/5))
- **`symbol` / `species` descriptions corrected** — across both tools' `symbol`, `species`, and `symbols` fields, the `ensembl_lookup_gene` top-level description, and the synced `CLAUDE.md`/`AGENTS.md` example, the wording no longer claims species is required for a symbol lookup; each states species defaults to `homo_sapiens` (override for other organisms). ([#5](https://github.com/cyanheads/ensembl-mcp-server/issues/5))
